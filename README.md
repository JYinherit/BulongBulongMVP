# Dangerous Token MVP

一款以情报传递为核心机制的多人桌游模拟器（网页版 MVP）。

---

## 本地运行

**前置要求：** Node.js

1. 安装依赖：
   ```bash
   npm install
   ```
2. 启动开发服务器：
   ```bash
   npm run dev
   ```

---

## 游戏模式

启动后，选择以下两种模式之一：

| 模式 | 说明 |
|------|------|
| **GM 模式** | 发牌员（GM）手动控制所有玩家发牌、情报操作与胜负判定。适合新手体验或主持人运营游戏。 |
| **随机模式** | ⚠️ **暂不可用**（功能尚在开发中）。计划支持系统自动分配阵营与自动发牌。 |

---

## 游戏规则

### 阵营与胜利条件

共有 **4 个阵营**，每局游戏由 GM 分配：

| 阵营 | 胜利条件 |
|------|----------|
| **THUMB（拇指）** | 任意一名 THUMB 玩家信物区集齐 **3 张绝密（红色）**情报 |
| **FINGER（食指）** | 任意一名 FINGER 玩家信物区集齐 **3 张珍贵（蓝色）**情报 |
| **BUS（巴士）** |  BUS 玩家信物区集齐 **6 张重要情报**（绝密+珍贵均可）或 隐藏任务完成 |
| **MYSTERY（神秘）** | 特殊阵营，胜利条件由 GM 裁定 |

### 情报属性

每张情报牌可拥有以下一个或多个属性：

| 属性 | 颜色标记 | 说明 |
|------|----------|------|
| 🔴 **绝密** | 红色圆点 | 对 THUMB 阵营至关重要 |
| 🔵 **珍贵** | 蓝色圆点 | 对 FINGER 阵营至关重要 |
| ⚫ **危险** | 黑色圆点 | 集齐 3 张危险情报将进入濒死状态 |

### 濒死与阵亡

- 玩家**信物区**中有 **3 张及以上危险情报**时，进入「濒死」状态。
- GM 可点击「确认阵亡」按钮使该玩家正式出局。
- 出局玩家的手牌与信物区情报将被销毁。

## 游戏阶段

每位玩家的回合按以下阶段依次进行：

1. **准备阶段（PREP）** — 系统自动跳过
2. **抽牌阶段（DRAW）** — GM 从控制台发牌给当前玩家，完成后点击「进入下一阶段」
3. **行动阶段（ACTION）** — 玩家可对手牌执行操作
4. **传递阶段（PASS）** — 玩家必须传递至少一张手牌（必需步骤）
5. **弃牌阶段（CLEANUP）** — 若手牌超过 6 张，需弃至 6 张
6. **结束阶段（END）** — 自动进入下一位玩家的回合

---

## 传递方式

在手牌上**双击**（GM 模式）或**悬停**（普通模式）可打开出牌菜单：

| 方式 | 说明 |
|------|------|
| **机密** | 情报面朝下传递，接收方只知道有情报被传来，不知内容 |
| **报告** | 情报公开传递，所有人可见内容 |
| **给予** | 指定一名玩家直接接收 |

目标玩家可选择**接收**或**拒绝**。若全部玩家拒绝，情报将**回到发起者信物区**（回旋飞镖规则）。

## GM 模式专属功能

- **发牌控制台**：选择目标玩家、设置情报标题与属性，点击「发给玩家」。
- **随机数生成器**：设置最小值/最大值，点击「掷骰」获得随机结果。
- **拖拽情报**：在 GM 模式下可拖拽任意情报至其他玩家手牌区或信物区。
- **双击操作菜单**：双击手牌可打开/关闭「机密/报告/给予」菜单（防误触拖拽）。
- **销毁情报**：点击情报右上角的 🗑️ 图标可销毁该情报。
- **修改阵营**：点击玩家名旁的阵营标签，可用下拉菜单更改阵营。
- **重命名玩家**：点击玩家名称可进行编辑。
